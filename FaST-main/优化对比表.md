### 优化1: GDP判断

#### 原代码
```python
def infer_gdp_level(desc):
    high_gdp_keywords = ["city", "urban", "hub", "central", "center"]
    desc_lower = str(desc).lower()
    for kw in high_gdp_keywords:
        if kw in desc_lower:
            return "High"
    return "Low"
```

**问题**: 
- 黄兴镇(长沙县, GDP 2129亿) → 可能被判为"Low"（无关键词）
- 嘉禾县(GDP 156亿) → 若描述含"county"可能被判为"High"

#### 优化后
```python
def parse_gdp_from_description(desc):
    pattern = r'GDP\s+is\s+([\d,]+\.?\d*)\s+billion\s+yuan'
    match = re.search(pattern, desc, re.IGNORECASE)
    if match:
        return float(match.group(1).replace(',', ''))
    return None

def infer_gdp_level_with_threshold(desc, gdp_threshold=1000.0):
    gdp_value = parse_gdp_from_description(desc)
    if gdp_value is None:
        return "Low"  # 降级方案
    return "High" if gdp_value >= gdp_threshold else "Low"
```

**效果**:
```
✅ 黄兴镇: GDP=2129.52亿 ≥ 1000亿 → High
✅ 嘉禾县: GDP=156.92亿 < 1000亿 → Low
✅ 长沙市: GDP=14331亿 ≥ 1000亿 → High
```

### 优化2: 动态误差阈值

#### 原代码
```python
ERROR_THRESHOLD_BY_FEATURE = {
    0: 100,  # 小客车上行
    1: 90,   # 小客车下行
    2: 27,   # 非小客车上行
    3: 27,   # 非小客车下行
}

need_correction = mae_per_sample > feature_threshold
```

**问题**:
| 场景 | 真实流量 | GNN预测 | 绝对误差 | 固定阈值 | 是否触发 | 合理性 |
|------|---------|---------|---------|---------|---------|--------|
| 高峰期 | 1200 | 1050 | 150 | 100 | ✅ 是 | ❌ 合理(12.5%误差) |
| 夜间 | 50 | 80 | 30 | 27 | ✅ 是 | ❌ 不合理(60%误差但仅差30辆) |
| 平峰期 | 400 | 480 | 80 | 100 | ❌ 否 | ❌ 应该触发(20%误差) |

#### 优化后
```python
def calculate_dynamic_threshold(true_values, feature_idx, base_threshold_pct=0.15):
    avg_true = np.mean(true_values)
    
    base_thresholds = {0: 80, 1: 80, 2: 40, 3: 40}
    base_thresh = base_thresholds.get(feature_idx, 60)
    
    # 相对误差阈值 = max(15% * 平均流量, 基准阈值 * 0.5)
    relative_thresh = max(avg_true * base_threshold_pct, base_thresh * 0.5)
    
    # 限制范围 [base_thresh * 0.5, base_thresh * 2.0]
    relative_thresh = np.clip(relative_thresh, base_thresh * 0.5, base_thresh * 2.0)
    
    return relative_thresh
```

**效果**:
| 场景 | 真实流量 | 动态阈值计算 | 最终阈值 | 绝对误差 | 是否触发 | 合理性 |
|------|---------|-------------|---------|---------|---------|--------|
| 高峰期 | 1200 | max(1200*15%=180, 80*0.5=40) | 180 | 150 | ❌ 否 | ✅ 合理(12.5% < 15%) |
| 夜间 | 50 | max(50*15%=7.5, 40*0.5=20) → clip到[20,80] | 20 | 30 | ✅ 是 | ✅ 合理(60%误差需修正) |
| 平峰期 | 400 | max(400*15%=60, 80*0.5=40) | 60 | 80 | ✅ 是 | ✅ 合理(20% > 15%) |

### 优化5: 分层采样

#### 原代码
```python
NORMAL_SAMPLE_RATIO = 0.1
norm_mask = np.random.rand(len(mae_per_sample)) < NORMAL_SAMPLE_RATIO
keep = np.where(need_correction | ev_mask | norm_mask)[0]
```

**问题**: 
- 所有样本统一10%采样率
- 可能导致高流量样本不足（因为数量少）
- 低流量样本过多（夜间数据多）

#### 优化后
```python
avg_true_per_sample = true_j.mean(axis=1)
q25, q75 = np.percentile(avg_true_per_sample, [25, 75])

# 低流量层 (0-25%): 5%采样率
low_flow_mask = avg_true_per_sample <= q25
norm_mask |= (low_flow_mask & (np.random.rand(len(mae_per_sample)) < 0.05))

# 中流量层 (25-75%): 15%采样率
mid_flow_mask = (avg_true_per_sample > q25) & (avg_true_per_sample <= q75)
norm_mask |= (mid_flow_mask & (np.random.rand(len(mae_per_sample)) < 0.15))

# 高流量层 (75-100%): 20%采样率
high_flow_mask = avg_true_per_sample > q75
norm_mask |= (high_flow_mask & (np.random.rand(len(mae_per_sample)) < 0.20))
```

**效果**:
假设某站点有1000个样本：
- 低流量(0-250): 250个 × 5% = 12.5个
- 中流量(250-750): 500个 × 15% = 75个
- 高流量(750-1000): 250个 × 20% = 50个
- **总计**: 137.5个正常样本（原方案仅100个）
- **优势**: 高流量样本占比从25%提升至36%，更均衡
