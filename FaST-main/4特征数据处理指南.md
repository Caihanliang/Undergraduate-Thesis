## 📋 4个特征定义

| 特征索引 | 特征名称 | 计算公式 |
|---------|---------|---------|
| 0 | 小客车上行 | 原始数据中的"小客车" + "上行" |
| 1 | 小客车下行 | 原始数据中的"小客车" + "下行" |
| 2 | 非小客车上行 | (汽车自然数 - 小客车) + "上行" |
| 3 | 非小客车下行 | (汽车自然数 - 小客车) + "下行" |

## 🔧 关键配置

### **模型配置 (HNGS_8_8_4FEAT.py)**
```python
MODEL = {
    "PARAM": {
        "num_nodes": 160,           # 站点数
        "num_features": 4,           # ⭐ 特征数从2改为4
        "input_len": 8,
        "output_len": 8,
        "channel_independent": True, # 通道独立处理
    },
    "FORWARD_FEATURES": [0, 1, 2, 3],  # 使用所有4个特征
    "TARGET_FEATURES": [0, 1, 2, 3],    # 预测所有4个特征
}
```

## ⚠️ 注意事项

1. **数据清洗**：
   - 自动过滤"断面"数据，只保留"上行"和"下行"
   - 非小客车 = 汽车自然数 - 小客车（确保非负）

2. **特征顺序**：
   - 必须严格按照：上行在前、下行在后
   - 小客车在前、非小客车在后

3. **时间对齐**：
   - 索引表示输入序列的**结束位置**
   - 预测起始时间 = 索引时间 - 7小时

4. **可视化**：
   - 4特征使用 2x2 子图布局
   - 左上：小客车上行
   - 右上：小客车下行
   - 左下：非小客车上行
   - 右下：非小客车下行

##  常见问题

### Q1: 数据形状不对？
**A**: 检查 his.npz 的形状应该是 `[时间步, 160, 4]`

### Q2: 训练报错维度不匹配？
**A**: 确认配置文件中 `num_features: 4`

### Q3: 可视化图只有2个子图？
**A**: 确保使用了新的可视化脚本 `可视化测试4特征.py`
